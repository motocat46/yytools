# slidingwindow 设计文档

## 设计目标

固定容量滑动窗口，支持 O(1) 的 Sum/Avg/Max/Min，新元素入窗时自动淘汰最旧元素。

---

## 核心结构选型

### Sum/Avg：增量维护

Sum 在每次 Add 时加上新值、减去被淘汰值，O(1) 维护无需扫描。Avg 直接用 sum/count 计算。

### Max/Min：单调双端队列

朴素实现每次 Max/Min 需要 O(N) 全窗口扫描，对大窗口代价显著（n=100k 时约 60,000 ns/op）。单调双端队列实现 O(1) 均摊：

- **单调递减队列**（Max）：队列始终保持从队首到队尾的值单调递减，队首即为当前窗口最大值。新元素入窗时，从队尾弹出所有小于等于新值的元素；旧元素出窗时，若队首序号等于被淘汰元素的序号则弹出队首。
- **单调递增队列**（Min）：对称结构，维护当前窗口最小值。

均摊 O(1) 的证明：每个元素至多入队一次、出队一次，总操作数 = 2N。

### 数据结构：环形缓冲区 + 逻辑序号

所有元素存储在固定大小的环形缓冲区 `buf[size]` 中，`wHead` 指向下一个写入位置。

单调队列不直接存储值，而是存储**逻辑序号**（`seq`），通过 `bufAt(seq)` 反查实际值。逻辑序号单调递增，避免在淘汰时判断队列中的序号是否"过期"——只需比较队首序号与被淘汰元素的序号是否相等。

### 为什么单调队列也使用环形缓冲区？

单调队列的容量上界等于窗口大小（最坏情况下每个元素都进队，严格单调时无弹出）。直接复用 `size` 作为单调队列容量，预分配 `maxBuf[size]` 和 `minBuf[size]`，避免动态扩容分配。

---

## 关键设计决策

### 空窗口操作 panic 而非返回零值

Max/Min/Avg 返回泛型 `T`，零值（0）可能是合法元素，返回零值是静默错误。采用 assert 强制调用方先用 `Empty()` 检查，将 bug 暴露在开发阶段。

### bufAt(seq) 通过逻辑序号反查值

物理索引计算：`idx = ((wHead - (seq - nextSeq)) % size + size) % size`，其中 `seq - nextSeq` 是该元素距最新写入位置的偏移。加 size 后取模避免 Go 取模运算对负数的语义问题。

## 复杂度

| 操作 | 时间 | 空间 |
|------|------|------|
| Add | O(1) 均摊 | O(1) |
| Sum | O(1) | O(1) |
| Avg | O(1) | O(1) |
| Max | O(1) 均摊 | O(1) |
| Min | O(1) 均摊 | O(1) |
| 整体空间 | — | O(N)，N 为窗口大小 |
