## 一、Floyd 采样算法（`SampleKDistinctFloyd`）

### 为什么选 Floyd 而不是 Knuth shuffle 截取

Knuth shuffle（Fisher-Yates）取前 k 个需要先构建完整数组（O(N) 空间），再随机打乱取前 k 个，适合 k 接近 N 的场景。

Floyd 采样只需 O(k) 空间，每次迭代只做一次随机操作，适合 k << N（从大范围中取少量样本）。

**Floyd 核心思路**：对 `j = N-k, N-k+1, ..., N-1`，在 `[0..j]` 取随机 `t`：
- 若 `t` 未被选中，选 `t`
- 否则选 `j`（j 保证未被选中）

每个元素被选中的概率均等，可以数学归纳证明。

## 二、`SampleWithMinGap` 的空间压缩技巧

**问题**：从 `[L..R]` 中采 k 个值，相邻值间隔 ≥ gap。

**直觉做法**：先采样，再检查间距，不满足则重试。最坏情况下重试次数不可控。

**空间压缩做法**（当前实现）：

设 `N = R - L + 1`，`M = N - (k-1)*gap`。

将问题转化为：在压缩后的 `[L..L+M-1]` 中用 Floyd 采 k 个不重复值，得到 `c[0..k-1]`（排序后），再按位置恢复间隔：

```
a[i] = c[i] + i * gap
```

**正确性直觉**：排好序的 `c` 中，相邻元素差 ≥ 1（不重复）。乘以 gap 后差 ≥ gap，满足约束。压缩保证了 `[L..L+M-1]` 内能放下 k 个不重复值（`M ≥ k`）。

**复杂度**：O(k) Floyd + O(k log k) 排序 = O(k log k)，无重试，确定性终止。

## 三、调用方负责传入 `*rand.Rand`

两个函数都要求调用方传入 `r *rand.Rand`，不内部创建。

**原因**：
1. 可复现性：调用方控制随机种子，方便测试和问题回放
2. 灵活性：调用方可选择全局随机源或局部固定种子
3. 与标准库惯例一致：`math/rand/v2` 的多数函数也接受 `*rand.Rand` 参数
